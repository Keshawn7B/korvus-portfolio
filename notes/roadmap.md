# Portfolio Roadmap

## Goal

Build a clean personal portfolio that makes your technical and creative work look real, organized, and impressive.

## Phase 1 — Inventory

- [ ] List existing coding projects
- [ ] List creative/media projects
- [ ] Pick 3 strongest projects
- [ ] Gather screenshots, links, and short descriptions
- [ ] Identify gaps worth filling with small new builds

## Phase 2 — Project writeups

Each project should answer:

- What is it?
- Why did you build it?
- What technologies did you use?
- What was hard or interesting?
- What would you improve next?
- Where can someone see the code/demo?

## Phase 3 — Site structure

Suggested pages:

- Home
- Projects
- About
- Skills
- Contact
- Lab / Experiments

## Phase 4 — Build

Good default stack:

- Astro or Next.js
- Tailwind CSS
- Markdown/MDX project entries
- Deploy to Vercel, Netlify, or GitHub Pages

## Phase 5 — Polish

- [ ] Mobile layout
- [ ] Project thumbnails
- [ ] GitHub links
- [ ] Live demo links
- [ ] SEO metadata
- [ ] Short bio
- [ ] Contact CTA

## First recommended project entry

Create a page for your Hermes/Korvus setup: multi-platform AI assistant, Notion/Gmail integrations, Keytana split profile, scheduled script-only reminders, and local Linux automation.
